### ADR-001
**[离线优先架构](./ADR-001-offline-first.md)**

**决策：** 使用 IndexedDB 实现前端离线存储

**影响：**
- ✅ 完全离线可用
- ✅ 隐私保护（数据不上传）
- ❌ 需要额外的数据同步机制

**相关文件：**
- `lib/idb.ts` - IndexedDB 接口
- `lib/storage.ts` - 存储管理

**参考阅读:**
- 下一章：[ADR-002 - 搜索策略](#adr-002)

### ADR-002
**[搜索索引缓存](./ADR-002-search-caching.md)**

**决策：** 实现智能缓存机制加速搜索

**关键点：**
- 使用 dirty flag 检测变更
- 一级缓存：内存（快）
- 二级缓存：IndexedDB（持久）

**性能提升：**
- 首次搜索：O(n)
- 缓存命中：O(1) - **10-20 倍加速**

**相关文件：**
- `lib/indexer.ts` - 搜索引擎
- `lib/storage.ts` - 缓存策略

**前置阅读：**
- 前一章：[ADR-001 - 离线优先](#adr-001)

**下一章：**
- [ADR-003 - AI 后端代理](#adr-003)

### ADR-003
**[AI 后端代理](./ADR-003-ai-backend-proxy.md)**

**决策：** 通过后端 API 代理 OpenAI 请求而非直接调用

**安全优势：**
- 🔒 API 密钥不暴露在浏览器
- 🔒 支持速率限制
- 🔒 完整审计日志

**架构：**
```
浏览器 → Next.js 后端 → Render 服务器 → OpenAI API
       ↑                  ↑
    前端安全           服务端安全
```

**相关文件：**
- `server/aiService.ts` - AI 服务
- `lib/aiService.ts` - 前端调用

**参考链接：** [DEPLOYMENT.md](../DEPLOYMENT.md) - 部署配置

### ADR-004
**[React Context](./ADR-004-react-context.md)**

**决策：** 使用 React Context 进行全局状态管理

**理由：**
- 避免 prop drilling 问题
- 内置于 React，无需额外库
- 足以支持当前应用规模

**使用场景：**
- 用户偏好（主题、语言）
- 全局通知
- UI 状态（侧边栏、模态框）

**相关文件：**
- `pages/_app.tsx` - 应用入口
