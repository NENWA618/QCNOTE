### ADR-005
**错误处理架构**

**决策：** 实现全局 React Error Boundary 组件处理应用级错误

**理由：**
- 提高应用稳定性，防止任何组件崩溃导致整体瘫痪
- 提供用户友好的错误页面和恢复选项
- 支持生产环境错误汇报

**实现方案：**
- `components/ErrorBoundary.tsx` 类组件
- 包装 _app.tsx 的主要内容
- 开发模式显示详细错误栈
- 生产模式显示简洁的用户界面

**错误处理流程：**
```
组件渲染错误 
  ↓
componentDidCatch() 捕获
  ↓
更新状态显示错误页面
  ↓
用户点击"刷新页面"恢复
```

**相关文件：**
- `components/ErrorBoundary.tsx` - 错误边界实现
- `pages/_app.tsx` - 集成点

### ADR-006
**后端安全防护**

**决策：** 实现速率限制和可选 API 密钥验证的中间件

**理由：**
- 防止 API 滥用和 DDoS 攻击
- 控制 OpenAI API 调用成本
- 支持多用户场景下的认证隔离

**实现方案：**

1. **速率限制中间件** (server/middleware.ts)
   - 限制：30 请求/分钟 per IP
   - 应用于：`/api/ai/*` 端点
   - 返回标准 HTTP 429 状态码
   - 自动清理过期条目防止内存泄漏

2. **API 密钥验证** (可选)
   - 启用条件：`REQUIRE_API_KEY=true` + 设置 `API_KEY`
   - 验证方式：`X-API-Key` 请求头
   - 失败响应：401 Unauthorized

**集成流程：**
```
请求 → rateLimitMiddleware（检查速率）
     → apiKeyMiddleware（检查密钥，可选）
     → 路由处理器（业务逻辑）
```

**性能头部信息：**
- `X-RateLimit-Limit: 30`
- `X-RateLimit-Remaining: 25`
- `X-RateLimit-Reset: 45`

**相关文件：**
- `server/middleware.ts` - 中间件实现
- `server/index.ts` - 集成配置

**相关参考：** [ARCHITECTURE.md](./ARCHITECTURE.md#security-middleware) 中的详细说明

### ADR-007
**图表性能优化**

**决策：** 采用多层次优化策略支持 1000+ 节点知识图谱渲染

**理由：**
- Canvas 渲染大规模图表时性能严重下降
- 用户需要在高缩放级别下探索知识图谱
- 力导向布局算法时间复杂度高 O(n²)

**优化策略：**

1. **节点聚类** - 按分类/密度分组相关节点
2. **LOD (细节层级)** - 缩放时动态调整显示精度
3. **视口虚拟化** - 仅渲染当前可见节点
4. **布局缓存** - 复用上次计算的位置
5. **简化模拟** - 减少迭代次数，使用高阻尼
6. **性能监控** - 跟踪帧率、渲染时间

**性能目标：**
- 100-500 节点：50-70% 加速
- 500-1000 节点：60-80% 加速
- 1000+ 节点：现可实用（启用全部优化）

**相关文件：**
- `lib/graphOptimization.ts` - 优化工具集
- `docs/GRAPH_OPTIMIZATION.md` - 实现指南

**相关参考：** [ARCHITECTURE.md](./ARCHITECTURE.md#graph-optimization) 中的详细说明
