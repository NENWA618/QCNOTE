## 常见事件类型

### 1. 数据库连接失败
**检测**: 监控警报 - DB连接数 > 80%
**影响**: 用户无法保存/加载笔记
**响应步骤**:
1. 检查PostgreSQL服务状态: `systemctl status postgresql`
2. 查看日志: `tail -f /var/log/postgresql/postgresql.log`
3. 重启服务: `systemctl restart postgresql`
4. 如果失败，检查磁盘空间: `df -h`
5. 联系DBA进行故障转移

**恢复时间目标 (RTO)**: 30分钟
**恢复点目标 (RPO)**: 1小时 (备份频率)

### 2. Redis缓存故障
**检测**: 应用错误日志 - Redis连接失败
**影响**: 推荐系统和会话管理受影响
**响应步骤**:
1. 检查Redis服务: `redis-cli ping`
2. 查看Redis日志: `tail -f /var/log/redis/redis.log`
3. 重启Redis: `systemctl restart redis`
4. 如果集群，检查主从同步
5. 清除应用缓存并重启应用

**RTO**: 15分钟
**RPO**: 无 (缓存数据)

### 3. API响应超时
**检测**: 监控 - p95响应时间 > 2秒持续5分钟
**影响**: 用户体验下降
**响应步骤**:
1. 检查应用日志中的错误
2. 监控CPU/内存使用: `top` 或 `htop`
3. 扩展实例 (如果使用容器): `docker scale qcnote=3`
4. 检查数据库查询性能
5. 如果是特定端点，禁用或回滚最近部署

**RTO**: 10分钟
**RPO**: 无

### 4. 安全事件
**检测**: 入侵检测系统警报
**影响**: 数据泄露风险
**响应步骤**:
1. 隔离受影响系统
2. 通知安全团队
3. 进行取证分析
4. 应用安全补丁
5. 通知受影响用户 (如果适用)
6. 更新事件响应计划

**RTO**: 立即
**RPO**: 根据事件
